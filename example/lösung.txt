# Analyse-Bericht: GreenMove E-Scooter Plattform

## 1. Zusammenfassung

Das Dokument beschreibt die Anforderungen für "GreenMove", ein E-Scooter-Verleihsystem für deutsche Großstädte. Das System besteht aus drei Hauptkomponenten: den Scootern (IoT-Hardware), einer Mobile App für Endkunden (iOS/Android) und einem webbasierten Verwaltungs-Dashboard. Kernfunktionen sind die Ausleihe per QR-Code, Standortverfolgung, automatische Abrechnung und Wartungsprozesse. Besonderer Wert wird auf Datenschutz (DSGVO), Skalierbarkeit und Compliance mit deutschen Gesetzen gelegt.

## 2. Funktionale Anforderungen

* **[REQ-F-01]** Das System muss den aktuellen Standort jedes Scooters alle 30 Sekunden per GPS empfangen und aktualisieren.
* **[REQ-F-02]** Das System muss Scooter mit einem Akkustand unter 10 % automatisch für neue Ausleihen sperren.
* **[REQ-F-03]** Das System muss Benutzern die Registrierung mittels E-Mail und Mobilfunknummer inklusive SMS-Verifizierung (2FA) ermöglichen.
* **[REQ-F-04]** Das System muss das Entsperren eines Scooters durch Scannen eines QR-Codes am Fahrzeug ermöglichen.
* **[REQ-F-05]** Das System muss Zahlungen per Kreditkarte, PayPal und SEPA-Lastschrift minutengenau abwickeln.
* **[REQ-F-06]** Das System muss dem Benutzer vor Fahrtantritt eine Sicherheitsunterweisung anzeigen, die bestätigt werden muss.
* **[REQ-F-07]** Das System muss verfügbare Scooter auf einer Karte anzeigen und eine kostenlose Reservierung (max. 15 Min.) erlauben.
* **[REQ-F-08]** Das System muss den Nutzer verpflichten, bei Fahrtende ein Beweisfoto des geparkten Scooters hochzuladen.
* **[REQ-F-09]** Das System muss das Beenden der Miete in definierten "No-Parking-Zonen" technisch unterbinden.
* **[REQ-F-10]** Das System muss ein Dashboard bereitstellen, das Status, Ladestand und Standort aller Scooter in Echtzeit anzeigt.
* **[REQ-F-11]** Das System muss für Ladepersonal ("Juicer") optimierte Einsammel-Routen berechnen.
* **[REQ-F-12]** Das System muss Administratoren ermöglichen, dynamische Preise basierend auf Wetter oder Nachfrage zu konfigurieren.
* **[REQ-F-13]** Das System muss bei Defektmeldungen automatisch ein Ticket im externen Ticketsystem (z.B. Jira) erstellen.

## 3. Nicht-Funktionale Anforderungen & Randbedingungen

* **[REQ-N-01]** **Datenschutz:** Alle personenbezogenen Daten müssen DSGVO-konform verarbeitet werden.
* **[REQ-N-02]** **Infrastruktur:** Der Serverstandort muss sich innerhalb der Europäischen Union befinden.
* **[REQ-N-03]** **Sicherheit:** Die Kommunikation (App-Server-Scooter) muss mittels TLS 1.3 verschlüsselt erfolgen.
* **[REQ-N-04]** **Performance:** Das System muss mindestens 10.000 gleichzeitige Ausleihvorgänge stabil verarbeiten.
* **[REQ-N-05]** **Latenz:** Die Antwortzeit der API darf bei Standardanfragen 200 Millisekunden nicht überschreiten.
* **[REQ-N-06]** **Kompatibilität:** Die App muss ab iOS 15 und Android 10 lauffähig sein.
* **[REQ-C-01]** **Hardware-Limit:** Die Geschwindigkeit der Scooter muss softwareseitig auf maximal 20 km/h begrenzt sein (gesetzliche Vorgabe).
